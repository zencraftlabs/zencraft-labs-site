Common Validation Examples

Maintain high data quality by implementing these validation patterns in your app configuration.

Custom ID (AAA-123)

^[A-Z]{3}-[0-9]{3}$

Perfect for internal reference numbers, hardware tags, or standardized project IDs.

Birth Date

^\d{2}\.\d{2}\.\d{4}$

Validates the standard European format: DD.MM.YYYY

IBAN

^[A-Z]{2}[0-9]{2}[A-Z0-9]{12,30}$

Ensures the correct length and structure of international bank account numbers.

Phone Number

^\+?[0-9\s\-]{7,15}$

Supports international formats including leading "+" and common separators.

Configuration

To apply these rules, update the rulesJson parameter in your Zendesk App Settings:

[
  {
    "fieldId": "12345",
    "regexPattern": "^[0-9]+$",
    "errorMessage": {
      "en": "Numbers only",
      "de": "Bitte nur Zahlen eingeben"
    }
  }
]
